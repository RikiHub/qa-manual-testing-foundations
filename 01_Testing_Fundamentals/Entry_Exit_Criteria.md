# Entry and Exit Criteria

## 1. Introducción

Este documento define los **criterios de entrada y salida** del proceso de testing para la aplicación web bajo prueba.

El objetivo es establecer **condiciones claras** que deben cumplirse antes de iniciar las pruebas y antes de considerar el ciclo de testing como finalizado, asegurando un proceso controlado y de calidad.

## 2. Criterios de Entrada (Entry Criteria)

Las actividades de testing podrán iniciar cuando se cumplan las siguientes condiciones:

- Requerimientos funcionales documentados y disponibles
- Criterios de aceptación definidos
- Ambiente de pruebas configurado y estable
- Acceso a la aplicación y credenciales de prueba
- Casos de prueba diseñados y revisados
- Datos de prueba disponibles
- Versión del sistema entregada para pruebas

Si alguno de estos criterios no se cumple, el inicio de las pruebas deberá ser evaluado y aprobado por las partes involucradas.

## 3. Criterios de Salida (Exit Criteria)

El ciclo de pruebas se considerará completado cuando se cumplan las siguientes condiciones:

- 100% de los casos de prueba ejecutados
- Defectos críticos y altos corregidos y verificados
- Defectos medios documentados y aceptados
- Evidencia de ejecución almacenada
- Test Summary Report elaborado y aprobado
- Riesgos residuales documentados y aceptados

## 4. Criterios de Suspensión y Reanudación

### 4.1 Criterios de Suspensión

Las pruebas podrán ser suspendidas si ocurre alguno de los siguientes escenarios:

- Ambiente de pruebas inestable
- Fallas críticas que impidan continuar las pruebas
- Cambios mayores en los requerimientos
- Versión del sistema no apta para pruebas

### 4.2 Criterios de Reanudación

Las pruebas se reanudarán cuando:

- El ambiente esté restablecido
- Los defectos bloqueantes hayan sido corregidos
- Se confirme la estabilidad de la nueva versión
- Se reevalúen los casos de prueba afectados

## 5. Beneficios de Definir Entry y Exit Criteria

- Control del proceso de testing
- Reducción de riesgos
- Mejor comunicación con stakeholders
- Decisiones objetivas y justificadas

## 6. Conclusión

La definición de criterios de entrada y salida contribuye a un proceso de testing **más estructurado, predecible y profesional**, asegurando que las pruebas se realicen bajo condiciones adecuadas y con resultados confiables.
