# Bug Report Template

## 1. Información General

| Campo | Descripción |
|-------|-------------|
| Bug ID | Identificador único del defecto |
| Título | Breve descripción del defecto |
| Módulo | Área afectada del sistema |
| Requerimiento | ID o descripción del requisito relacionado |
| Reportado por | QA Analyst |
| Fecha | Fecha de detección |
| Prioridad | Alta / Media / Baja |
| Severidad | Crítica / Alta / Media / Baja |
| Estado | Nuevo / En progreso / Resuelto / Cerrado |

## 2. Descripción Detallada

- **Resumen:** Explicación clara del problema.  
- **Comportamiento Actual:** Qué hace el sistema actualmente.  
- **Comportamiento Esperado:** Qué debería hacer el sistema.

## 3. Pasos para Reproducir

1. Paso 1 – Acción a realizar  
2. Paso 2 – Acción siguiente  
3. Paso 3 – Resultado observado

> Nota: Deben ser claros y replicables por cualquier tester o desarrollador.

## 4. Datos de Prueba

- Información utilizada (usuario, contraseña, datos específicos, navegador, etc.)  
- Configuración del sistema / ambiente de pruebas

## 5. Evidencia

- Capturas de pantalla o video  
- Logs relevantes si aplica

## 6. Observaciones / Comentarios Adicionales

- Información adicional para el equipo de desarrollo  
- Impacto sobre el usuario o flujo del negocio

## 8. Consejos de Buenas Prácticas

- Títulos claros y concisos  
- Pasos reproducibles y en orden  
- Indicar severidad y prioridad de manera objetiva  
- Adjuntar evidencia siempre que sea posible  
- Evitar información irrelevante o ambigua
