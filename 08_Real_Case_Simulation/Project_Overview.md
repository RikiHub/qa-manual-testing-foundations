# Real Case Simulation – QA Manual Testing Project

## 1. Introducción

Este documento representa una **simulación de un proyecto real de QA Manual**, cuyo objetivo es demostrar cómo un QA Analyst aborda un proyecto de principio a fin.

La finalidad de esta simulación es evidenciar:
- Planeación y estrategia de pruebas
- Diseño de casos de prueba
- Ejecución de pruebas manuales
- Reporte y seguimiento de bugs
- Análisis de cobertura, riesgos y resultados
- Recomendación final para liberación del producto

El proyecto simula una **aplicación web de gestión de usuarios**, un escenario común en productos reales.

## 2. Application Under Test (AUT)

**Nombre de la aplicación:** Sistema de Gestión de Usuarios  
**Tipo:** Aplicación Web  
**Funcionalidades principales:**
- Registro de usuarios
- Inicio de sesión
- Gestión de sesión
- Validaciones de campos
- Manejo de mensajes de error

**Target Users:** Usuarios finales que acceden a una plataforma segura

## 3. QA Role y Responsabilidades

En esta simulación, el rol de QA Analyst es responsable de:

- Analizar y comprender los requerimientos del negocio
- Definir la estrategia y el plan de pruebas
- Diseñar casos de prueba manuales
- Ejecutar casos de prueba
- Reportar y dar seguimiento a bugs
- Analizar cobertura y riesgos
- Emitir una recomendación final sobre la liberación del sistema

## 4. Alcance de las Pruebas

### In Scope
- Funcionalidad de inicio de sesión
- Funcionalidad de registro de usuarios
- Validaciones de campos de entrada
- Mensajes de error y retroalimentación al usuario
- Validaciones básicas de seguridad (enfoque manual)

### Out of Scope
- Pruebas de rendimiento
- Pruebas automatizadas
- Pruebas de seguridad avanzadas (pentesting)
- Pruebas de backend o APIs

## 5. Supuestos

- Los requerimientos se consideran estables durante el ciclo de pruebas
- El ambiente de pruebas está disponible y funcional
- Las pruebas se realizan de forma manual
- Los datos de prueba están predefinidos y controlados

## 6. Entregables de QA

Durante esta simulación se generan los siguientes artefactos de QA:

- Estrategia de pruebas
- Plan de pruebas
- Casos de prueba
- Reportes de bugs
- Reporte de ejecución de pruebas
- Recomendación final de QA

Todos los entregables se documentan dentro de este repositorio.

## 7. Objetivo de la Simulación

El objetivo de esta simulación es demostrar:

- Un flujo completo de trabajo de QA en un proyecto real
- Buenas prácticas en documentación de pruebas
- Capacidad de análisis y toma de decisiones
- Preparación para participar en proyectos reales como QA Analyst
