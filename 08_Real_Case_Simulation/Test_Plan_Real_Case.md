# Plan de Pruebas – Simulación de Caso Real QA Manual

## 1. Introducción

Este **Plan de Pruebas** describe las actividades, recursos, alcance y cronograma necesarios para ejecutar las pruebas manuales del proyecto **Sistema de Gestión de Usuarios**.

El plan se basa en la estrategia definida previamente y establece cómo se llevarán a cabo las pruebas para asegurar la calidad del producto antes de su liberación.

## 2. Objetivo del Plan de Pruebas

- Definir claramente el alcance de las pruebas
- Establecer responsabilidades y actividades
- Planificar la ejecución de pruebas manuales
- Garantizar una correcta cobertura de los requerimientos
- Proveer información para la decisión de liberación del sistema

## 3. Alcance

### Funcionalidades a probar
- Registro de usuarios
- Inicio de sesión
- Validaciones de campos
- Mensajes de error
- Manejo básico de sesión

### Funcionalidades fuera de alcance
- Pruebas automatizadas
- Pruebas de rendimiento
- Pruebas de seguridad avanzadas
- Pruebas de APIs y backend

## 4. Enfoque de Pruebas

- Pruebas manuales funcionales
- Pruebas positivas y negativas
- Pruebas exploratorias
- Pruebas de regresión manual
- Smoke testing previo a cada ciclo

## 5. Entregables

- Casos de prueba documentados
- Evidencias de ejecución
- Reportes de bugs
- Reporte de ejecución de pruebas
- Reporte final de QA

## 6. Ambiente de Pruebas

- **Ambiente:** QA / Staging
- **Navegadores:** Chrome, Edge
- **Sistema Operativo:** Windows 11
- **Datos de prueba:** Usuarios válidos, inválidos y casos límite

## 7. Roles y Responsabilidades

| Rol | Responsabilidad |
|-----|-----------------|
| QA Analyst | Diseño y ejecución de pruebas, reporte de bugs |
| Developer | Corrección de bugs |
| Product Owner | Validación y aprobación |

## 8. Criterios de Entrada

- Requerimientos aprobados
- Ambiente disponible y estable
- Casos de prueba revisados
- Datos de prueba preparados

## 9. Criterios de Salida

- 100% de casos críticos ejecutados
- No existen bugs críticos abiertos
- Bugs medios evaluados
- Reporte final de pruebas aprobado

## 10. Cronograma de Pruebas (Estimado)

| Actividad | Duración |
|-----------|----------|
| Análisis de requerimientos | 1 día |
| Diseño de casos de prueba | 2 días |
| Ejecución de pruebas | 3 días |
| Reporte de bugs | Paralelo |
| Re-ejecución / regresión | 1 día |
| Reporte final | 1 día |

## 11. Riesgos y Contingencias

| Riesgo | Acción |
|--------|--------|
| Cambios de requerimientos | Revisión y ajuste del plan |
| Retrasos en correcciones | Repriorización de pruebas |
| Bugs críticos | Bloqueo del release |
| Falta de tiempo | Enfoque en pruebas críticas |

## 12. Aprobación

Este plan de pruebas sirve como base para la ejecución de pruebas manuales y la toma de decisiones relacionadas con la liberación del sistema.
